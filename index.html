DevOps CI/CD Portfolio Project

Automated Build, Push, and Deployment
with Jenkins and Docker

This project demonstrates a practical DevOps workflow where application changes are pushed to GitHub, processed through a Jenkins CI/CD pipeline, containerized with Docker, and deployed in a repeatable and reliable way.

Deployment Successful Version: v1.0.0

Source Control

Code is managed in GitHub with version tracking and commit-based workflow for structured collaboration and change history.

CI/CD Automation

Jenkins automates build and deployment steps, reducing manual work and improving consistency across releases.

Containerization

Docker packages the application into a portable container image so it can run reliably across environments.

Application Status
Live and Running
Environment
Local / Docker / Jenkins
Pipeline Flow
  • Step 1: Developer pushes code to GitHub
  • Step 2: Jenkins triggers the pipeline
  • Step 3: Docker image is built
  • Step 4: Image is pushed to Docker Hub
  • Step 5: Application is deployed and verified

Built as a hands-on DevOps project to demonstrate practical experience with Git, Jenkins, Docker, CI/CD pipelines, and automated deployment workflows.
